Offensive and Defensive Cybersecurity Challenges

Academic Year 2025/2026

About This Project

This repository contains solved challenges from the Offensive and Defensive Cybersecurity course at Politecnico di Milano, Academic Year 2025/2026.

These challenges cover various aspects of offensive and defensive cybersecurity, including binary exploitation, reverse engineering, and security mitigation techniques. Each challenge folder contains the challenge binaries, exploit scripts, and solutions developed during the course.

Topics Covered

  • Shellcode - Writing and exploiting shellcode
  • Reverse Engineering - Binary analysis and reverse engineering techniques
  • Mitigation Techniques - Understanding and bypassing security mitigations
  • Return-Oriented Programming (ROP) - ROP chain construction and exploitation
  • Symbolic Execution - Automated program analysis using symbolic execution
  • Heap Exploitation - Heap-based vulnerabilities and exploitation techniques
  • Kernel Exploitation - Linux kernel security and exploitation
  • Packing and Obfuscation - Analyzing packed and obfuscated binaries
  • Race Conditions - Exploiting race condition vulnerabilities
  • Final CTF - Capture The Flag competition challenges

Repository Structure

The repository is organized into folders for each challenge topic:

  • 01-Shellcode/ - Shellcode writing challenges
  • 02-Reverse/ - Reverse engineering challenges
  • 03-Mitigation/ - Security mitigation bypass challenges
  • 04-ROP/ - Return-Oriented Programming challenges
  • 05-SymbolicExec/ - Symbolic execution challenges
  • 06-Heap/ - Heap exploitation challenges
  • 07-Kernel/ - Kernel exploitation challenges
  • 08-Packing/ - Binary packing and obfuscation challenges
  • 09-RaceCondition/ - Race condition exploitation challenges
  • Final_CTF/ - Final CTF competition challenges

Each challenge folder contains the challenge binaries, exploit scripts, and solutions developed during the course.

Tools and Environment

The challenges were solved using various tools and techniques commonly used in offensive security, including:

  • Debuggers: GDB, radare2
  • Disassemblers: IDA, Ghidra
  • Exploitation Frameworks: pwntools, libdebug
  • Analysis Tools: angr, z3

Be sure to have created the virtual environment with those libraries!

Note: This repository documents my personal experiments and attempts at solving these challenges. As such, it may contain errors or suboptimal solutions. Please treat the contents as a reference rather than a definitive guide.
